# Launch Checklist

**Feature/Product:** [Name]
**Launch date:** [Date]
**Launch tier:** Tier 1 / 2 / 3
**PM owner:** [Name]
**Marketing owner:** [Name]

---

## T-4 Weeks

### Product
- [ ] Feature complete and passing QA
- [ ] Feature flagged — can be enabled/disabled per cohort
- [ ] Analytics events instrumented and verified
- [ ] Edge cases handled: empty state, error state, loading state
- [ ] Accessibility audit passed

### Marketing
- [ ] Launch brief shared with Marketing and PMM
- [ ] Target segment and messaging aligned
- [ ] Blog post / launch copy in draft

### Sales & CS
- [ ] Launch brief shared with Sales and CS
- [ ] Feature impact on current customers identified

---

## T-2 Weeks

### Product
- [ ] Beta feedback incorporated
- [ ] Performance benchmarks met
- [ ] Rollback plan documented and tested

### Marketing
- [ ] Blog post approved
- [ ] Email draft approved
- [ ] In-product announcement (if applicable) designed

### Sales
- [ ] Competitive battlecard updated
- [ ] Sales training scheduled
- [ ] Demo environment updated

### CS / Support
- [ ] Support documentation written
- [ ] CS team briefed on what's changing and common questions
- [ ] In-app help text reviewed

---

## T-1 Week

- [ ] All teams confirmed ready
- [ ] Rollout plan confirmed: % → % → 100%
- [ ] Monitoring dashboard ready — who's watching what on launch day?
- [ ] Rollback decision tree documented: what signals trigger a rollback?
- [ ] On-call plan for launch day confirmed

---

## Launch Day

- [ ] Feature flag enabled per plan
- [ ] Analytics confirmed firing correctly
- [ ] No critical errors in first hour
- [ ] Blog post published (Tier 1/2)
- [ ] Email sent to target segment (Tier 1/2)
- [ ] Sales notified it's live

---

## T+1 Week

- [ ] Initial adoption metrics reviewed
- [ ] Support ticket volume reviewed — any unexpected spikes?
- [ ] Rollout complete to 100% (if staged)
- [ ] Any critical issues documented and addressed

---

## T+4 Weeks

- [ ] Post-launch review completed
- [ ] Success metric vs. target reviewed
- [ ] Qualitative feedback reviewed (reviews, support, CS feedback)
- [ ] Learnings documented
- [ ] Next steps decided: build out / iterate / redirect
