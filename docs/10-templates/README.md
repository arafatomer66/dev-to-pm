# Templates

Copy, paste, and adapt. These are working templates — not theoretical frameworks.

## Available templates

| Template | What it's for | When to use it |
|----------|--------------|----------------|
| [PRD](prd.md) | Full product requirements document | Medium-to-large features |
| [Product Brief](product-brief.md) | Lightweight one-pager | Small features, quick alignment |
| [Roadmap](roadmap.md) | Now/Next/Later roadmap | Quarterly planning, stakeholder communication |
| [Competitive Snapshot](competitive-snapshot.md) | One-page competitor overview | Competitive intelligence, sales enablement |
| [Metrics Framework](metrics-framework.md) | North Star + input metrics | OKR setting, launch preparation |
| [Launch Checklist](launch-checklist.md) | Pre-launch readiness | Feature and product launches |
| [Stakeholder Map](stakeholder-map.md) | Who needs what from you | Project kickoffs, major initiatives |
| [Decision Log](decision-log.md) | Record of key decisions and rationale | Any significant product decision |
| [Experiment Canvas](experiment-canvas.md) | Hypothesis + test design | Discovery validation, A/B tests |
| [Status Update](status-update.md) | Progress communication | Weekly/monthly stakeholder updates |
| [User Research Guide](user-research-guide.md) | Interview script + synthesis | Customer discovery, usability research |

## When to use what

**Starting a new initiative:** Product Brief → PRD → Metrics Framework → Roadmap

**Competitive situation:** Competitive Snapshot → Stakeholder Map (who needs to know)

**Running experiments:** Experiment Canvas → Metrics Framework

**Weekly operations:** Status Update → Decision Log (when a decision is made)

**Team onboarding:** Stakeholder Map → Roadmap

## A note on templates

Templates are starting points, not constraints. Every team has different vocabulary, different processes, and different stakeholders. Adapt these to fit. The goal is to have a consistent structure that speeds up writing and makes documents predictable — not to enforce bureaucracy.

The worst outcome is a PM who spends more time filling in template sections than thinking about the product. If a section of the template doesn't apply, delete it.
